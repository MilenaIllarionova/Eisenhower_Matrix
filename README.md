# Matrix — веб-приложение для совместного планирования задач с матрицей Эйзенхауэра

Практическая часть ВКР по теме «Разработка web-приложения для совместного планирования задач с матрицей Эйзенхауэра».

Две независимые папки:

- **`matrix-server/`** — Node.js + Express + TypeScript, MongoDB, JWT, Socket.IO.
- **`matrix-client/`** — Vite + React 18 + TypeScript, Tailwind, Zustand, dnd-kit, date-fns.

В каждой папке есть свой `README.md` с эндпоинтами, структурой и инструкцией.

## Реализованные экраны и поведения

1. **Вход / Регистрация** — split-layout, бренд «Matrix», полностью на русском.
   - Анимированный «язычок»: тёмная карточка плавно перемещается между вкладками «Вход» и «Регистрация».
   - Активная вкладка подсвечивается белым текстом.
2. **Доски (главный экран «Привет!»)**
   - Поиск (TopBar) — по доскам, задачам и участникам. У участника показывается список его задач, клик ведёт на задачу.
   - Аватар в шапке → переход на страницу профиля.
   - Карточка доски кликабельна → переход на матрицу с фильтром по этой доске (`/tasks?boardId=...`).
   - «+» в блоке досок открывает модалку «Добавить новую доску» (название, описание, поиск участников с ролями admin / member / viewer).
   - «+» в блоке «Крайний срок» открывает модалку задачи, где «Срок» **автоматически** установлен на сегодня.
   - «+» в блоке «Друзья» открывает модалку «Добавить в друзья» (поиск участников + приглашение по email).
3. **Сроки (календарь)**
   - Переключатель **Месяц / Неделя / День** с плавающим белым индикатором (та же анимация, что и в сайдбаре).
   - **Month** — сетка 7 × N с задачами-капсулами; **Week** — 7 дней × часы с цветными блоками; **Day** — список часов с подробной карточкой задачи.
   - Клик по задаче в любом представлении ведёт на матрицу нужной доски.
4. **Матрица Эйзенхауэра (Задачи)**
   - Фильтр по статусам: Предстоящие / В процессе / На проверке / Выполнено.
   - Участники доски (подтягиваются с сервера по `boardId`).
   - 4 квадранта; перетаскивание между ними через `@dnd-kit`, оптимистичный апдейт + REST.
   - Клик по карточке → **детальная модалка задачи**: название, описание, исполнитель (с делегированием), срок (дата + время), статус, кнопки **Отмена / Сохранить / Выполнено**.
   - **Делегирование**: выбор нового исполнителя в отдельной модалке-пикере → ему отправляется уведомление (Socket.IO + тост).
   - **Выполнено**: задача меняет статус на `review` → владелец доски получает уведомление о необходимости проверки.
5. **Профиль** — карточка пользователя, его доски и активные задачи; кнопки «Редактировать» и «Выйти».
6. **Тосты-уведомления** — всплывают слева снизу (по макету: тип «Завершение задачи», «Новая задача» и т.д.). Подписываются на сокет-событие `notification:new` и отображаются автоматически.

## Архитектура (важные изменения по правкам)

### Бэкенд

- В `Task.status` добавлен новый литерал `'review'` (на проверке у владельца доски).
- В `NotificationType` добавлены: `task_review_requested`, `task_completed`, `project_invited`.
- Новая модель `ProjectMember` (`{ projectId, userId, role: 'admin' | 'member' | 'viewer', joinedAt }`) с уникальным индексом `(projectId, userId)`.
  При создании доски её владелец автоматически становится `admin`. Доступ к задачам доски теперь проверяется через `ProjectMember`.
- `project.service`:
  - `createProject` принимает `memberInvites` (массив `{ userId, role }`) — сразу добавляет участников с нужными ролями и шлёт им `project_invited`-уведомления;
  - `addProjectMember(projectId, adminId, email, role)` — приглашение по email с проверкой админ-прав;
  - `assertProjectMembership`, `assertProjectAdmin` — guard-функции.
- `task.service.updateTask`:
  - смена `assigneeId` → уведомление `task_assigned` новому исполнителю;
  - смена `status → 'review'` → уведомление `task_review_requested` владельцу доски;
  - смена `status → 'done'` (после проверки) → уведомление `task_completed` исполнителю.

### REST API (новое и изменённое)

| Метод  | Путь                              | Описание                                         |
|--------|-----------------------------------|--------------------------------------------------|
| GET    | /api/projects                     | Список досок, где пользователь — участник       |
| POST   | /api/projects                     | Создание доски + опциональный `memberInvites`   |
| GET    | /api/projects/:id/members         | Состав участников доски (с ролями)              |
| POST   | /api/projects/:id/invite          | Пригласить участника по email с ролью           |
| PATCH  | /api/projects/:id                 | Редактирование (admin)                          |
| PATCH  | /api/tasks/:id                    | `status: 'review'`/`'done'` запускает workflow  |

Полный список эндпоинтов — в `matrix-server/README.md`.

### Фронтенд

- `src/i18n/ru.ts` — централизованные строки и функции форматирования дат на русском.
- `ViewToggle` — общий компонент со sliding-индикатором, используется на Boards, Deadline и Matrix.
- `Sidebar` — анимированный тёмный «язычок» (`useLayoutEffect` + `transition-all`) перемещается между активными пунктами меню.
- `AuthLayout` — тот же приём для табов «Вход» / «Регистрация».
- `SearchBar` — поиск по доскам, локальным задачам и серверным пользователям; результаты сгруппированы.
- `BoardFormModal` — создание доски, поиск участников с выбором роли, кнопки «Сохранить»/«Отмена».
- `TaskFormModal` — новый дизайн по макету; флаг `burning` подставляет сегодняшнюю дату.
- `AddFriendModal` — поиск + приглашение по email.
- `TaskDetailModal` — раскрытие задачи (название, описание, исполнитель, срок дата+время, статус, кнопки **Отмена / Сохранить / Выполнено**).
- `AssigneePickerModal` — выбор исполнителя из участников доски, поиск, radio-индикация.
- `ToastHost` + `useToastStore` — гирлянда тостов слева снизу; подписана на `notification:new` через Socket.IO.
- `ProfilePage` — карточка пользователя со списками досок и задач.
- Календарь: три отдельных компонента `CalendarGrid` (месяц), `CalendarWeek` (неделя × часы), `CalendarDay` (часы). Клик по задаче — навигация в матрицу нужной доски через `useNavigate`.

## Быстрый старт

```bash
# 1. Сервер
cd matrix-server
cp .env.example .env       # MONGODB_URI, JWT_SECRET
npm install
npm run dev                # http://localhost:4000

# 2. Клиент
cd matrix-client
npm install
npm run dev                # http://localhost:5173 (проксирует /api и /socket.io на :4000)
```

Требуется работающий MongoDB (локально через `mongod` или MongoDB Atlas).
